The primitive types in JavaScript are:

  • number - a 64 bit floating point
  • boolean
  • string
  • null
  • undefined
  • symbol

The other data types include, but are not limited to:

  • Object
  • Function
  • Array
  • Date
  • Error

There are two ways to check for equality in JavaScript:

  1. Double Equals => a relational operator that returns true if and only if both operands have the same value. Type coersion is performed with this operator. (!= will also type coerce)
  2. Triple Equals => a relational operator that returns true if and only if both operands are both of the same type and have the same value. Type equality is checked first. (!== will not type coerce.)

Truthy and Falsy

"Falsy" refers to any value that is not the boolean false, however, it evaluates to false.
Values that are "falsy" include:

  • The empty string ''
  • The number 0
  • null and undefined
  • NaN

Null and undefined are not equal to anything but themselves.
NaN is not equivalent to anything, including itself.

Every other value is truthy. If it's technically not true, false, or falsy, it's truthy.

We can use the 'typeof' operator to determine what type a certain value is. The returned operand is a string of the type of data.



The different variable scopes in JavaScript are:

  • Global - Anything not in the local or block scope. Accessible throughout the entire program.
  • Local - Anything within a function's scope/block of code.
  • Block - Anything within a block.

Functions in JavaScript are special.

  • They can be assigned in one of three ways:
    • As a Declared function
    • Function as a value
    • As a Arrow function
  • They are Objects
  • They can be anonymous (nameless)
  • They can be hoisted!
  • They can be self-invoking:
    Self invoking function syntax is to wrap a function in a set of parenthesis,", "and have a trailing set of parenthesis for any parameter
  • They can be used as a callback function (They can be passed as a parameter into another function)

New in EcmaScript 6:

  • Template Literals: strings created using ES6 backticks, ``, which allow for the interpolation of JavaScript expressions inside strings.
  • Arrow Functions: provide shorter syntax than the Declaration of a Function. Arrow functions are necessary because they inherit the 'this' value of its enclosing context.
