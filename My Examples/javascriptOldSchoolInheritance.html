As we know, Inheritance is achieved in JavaScript through Prototypal Inheritance.

  • Every object has a prototype. And every prototype has a prototype. So on, and so forth, until it reaches Object.prototype
  • The very top of the Prototypal Inheritance chain? Believe it or not, null
  • When a property gets accessed on an object in JavaScript (either state or behavior), JS looks on the object for that property first.
  • If the property is not found on the object, JS checks the object's prototype.
  • If the property is not on the object's prototype, JS checks the prototype of the prototype, and so on
  • If we want to add state or behavior to an object, we have to add it to the object's prototype
